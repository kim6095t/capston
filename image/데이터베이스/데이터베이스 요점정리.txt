database
-논리적으로 연관된 데이터를 모아 일정한 형태로 저장해 놓은것

dbms(database management system(dbms)
-데이터베이스 관리 프로그램
-dbms를 이용하여 데이터 입력, 수정, 삭제등의 기능 제공

데이터베이스 특징
-통합된 데이터: 중복으로 인한 데이터 불일치 현상 제거
-저장된 데어터: 컴퓨터가 접근할 수 있는 저장매체에 저장된 데이터
-운영 데이터: 조직의 목적을 위해 사용하는 데이터
-공용 데이터: 여러업무에 사용함

dbms 필수기능
데이터 정의, 데이터 조작, 데이터 제어

dbms장점
-데이터 중복의 최소화
-데이터의 공용
-일관성 유지
-무결성 유지
-보안 보장
-표준화 용이
-전체 데이터 요구의 조정

dbms단점
-운영비 증대
-특정 응용 프로그램의 복잡화
-복잡한 백업과 회복
-시스템 취약성 ex)장애가 일어나면 업무가 중단




스키마- db내의 데이터의 구조, 관계, 제약조건에 대한 명세
외부스키마(서브스키마)- 사용자 뷰, 여러개 존재가능
내부스키마-데이터베이스의 물리적 구조를 나타냄(실제 저장방법 기술)
개념스키마- 전체적인뷰, 개체간의 관계와 제약조건, 외부스키마 통합

관계형 데이터베이스의 특징
-2차원 테이블로 표현
-데이터 무결성
-데이터 처리를 위해 sql 사용
-한번에 레코드를 처리하는 것이 아니라 집합을 처리함





릴레이션=테이블=실질적으로 자료가 저장되는 집합체
-릴레이션 스키마(속성들의 집합)+릴레이션 인스턴스(튜플의 집합)
-속성(로우),튜플(컬럼)으로 이뤄져있다
-차수: 속성의 갯수, 카디널러티: 튜플의 갯수

릴레이션 특징
-튜플의 상이성: 모든 속성의 값이 동일한 튜플 존재x
-튜플의 무순서성
-속성의 무순서성
-속성의 원자성: 값이 없거나 딱하 나만 있어야한다

제약조건
-중복확인, 빈칸못쓰게 하는거 등등

ERD-테이블을 어떻게 관리해야지 잘 관리할수있을까 고민하고 나오는 설계도



.
키: 튜플을 항상 유일하게 식별할 수 있는 속성들의 집합
후보키: 유일성하고 최소성 만족하는 키
슈퍼키: 유일성은 만족하나 최소성은 만족 못하는거

기본키:ex)사번
-후보키중 설계자가 특별히 지정한 키
-널값 불가능(개체 무결성 제약)
# 여기에 해당되지 않은 키는 대체키

외부키: ex)부서코드
-다른코드에 있는지 확인하는 안전장치같은 역할
-NULL 값 가능
-다른 릴레이션의 기본키
- 참조할 수 없는 외래키는 가져서는 안됨(참조 무결성 제약)




관계대수
-절차언어(원하는 데이터를 어떻게 찾는지에 대한 성격이 강함)
-일반 집합연산자와 순수 관계 연산자 두 종류가 있다
https://chartworld.tistory.com/12

일반 집합연산자
• 합집합 (union, ∪)
• 교집합 (intersect, ∩)
• 차집합 (difference, −)
• 카티션 프로덕트 (cartesian product, ⅹ)

순수 관계 연산자 (Relational operator)
• 실렉트 (select, σ)
• 프로젝트 (project, Π)
• 조인 (join, ⋈)
 -세타조인:비교연산자(=,>,<)를 이용하여 표현된 조인
 -동일조인:비교연산자가 =인 조인
 -자연조인:중복된 속성 제거한 조인
• 디비전 (division, ÷ )

기본연산: 합집합, 차집합, 카티션 프로덕트, 실렉트, 프로젝트
복합연산: 조인, 교집합, 디비전

-세미조인
-외부조인: 대응되는 값이 없어도 조인
-외부 합집합: 스키마가 다르더라도 합집합이 가능


관계해석
-비절차 언어(어떤조건인지만 알려주면 연산함) 
-기능적으로는 관계 대수와 동등







sql:관계형 데이터베이스에서 데이터조작과 정의하기 위해 사용되는 언어
sql 작업
-데이터 조회
-테이블에서 행 삽입, 갱신 및 삭제
-객체 생성, 변경 및 삭제
-데이터베이스 및 해당 객체에 대한 엑세스제어
-데이터 베이스 일관성 및 무결성 보장

DML데이터 조작어(select, insert, update, delete, merge)
DDL데이터 정의어(create, alter, drop, rename, truncate)
TCL(commit, rollback)
DCL데이터 제어어(grant, revoke)
